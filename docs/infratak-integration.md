# infra-TAK integration profile

This document describes how [infra-TAK](https://github.com/takwerx/infra-TAK) deploys the EUD Remote Assist Portal without modifying tracked files in the git clone. After upstream **v2.1.0+**, updates are:

```bash
cd ~/eud-remote-assist
git pull --ff-only
docker compose up -d --build
```

`git status` should remain clean (only gitignored `.env` differs).

Related: [versioning.md](versioning.md), [mdm-config.md](mdm-config.md), [manual-install.md](manual-install.md)

---

## Integration model

infra-TAK is a host orchestrator. For EUD Remote Assist it:

1. Clones `https://github.com/cfd2474/EUD_Remote_Assist_Portal.git` â†’ `~/eud-remote-assist`
2. Writes a **gitignored** `.env` with site-specific values (FQDN, Authentik OIDC, passwords)
3. Provisions an **Authentik OAuth2 provider** (slug `eud-remote-assist`, public SPA + PKCE)
4. Runs **Docker Compose** from the clone (no edits to `docker-compose.yml`)
5. Fronts the stack with **Caddy** (TLS termination, port split)
6. On update: `git pull --ff-only` â†’ refresh `.env` â†’ `docker compose up -d --build`

## Port split (Caddy)

| Port | Traffic |
|------|---------|
| **443** (`https://remote.<fqdn>`) | Admin portal + OIDC only |
| **8448** (`https://remote.<fqdn>:8448`) | Device API: `/api/v1/*`, `/ws/device`, `/health`, `/version` |

Caddy blocks device paths on 443 and proxies them only on 8448.

```
Internet
   â”‚
   â”śâ”€ :443  Caddy â†’ 127.0.0.1:8767  (admin; blocks /api/v1, /ws/device, /health, /version)
   â”‚
   â””â”€ :8448 Caddy â†’ 127.0.0.1:8767  (device API paths)

Docker nginx (127.0.0.1:8767 â†’ container :80) â†’ server:3000 + web:80
```

- **UFW:** infra-TAK allows `8448/tcp` publicly; denies direct access to `8767`
- **No host exposure** of postgres or Node server ports
- Inner nginx is **HTTP only** on loopback

## Docker nginx binding (no compose patch)

`docker-compose.yml` publishes container port 80 using:

```yaml
ports:
  - "${NGINX_BIND_ADDR:-}:${HTTP_PORT:-80}:80"
```

| Profile | `NGINX_BIND_ADDR` | `HTTP_PORT` | Result |
|---------|-------------------|-------------|--------|
| Standalone (default) | *(empty)* | `80` | `0.0.0.0:80` |
| Host nginx upstream | `127.0.0.1` | `8091` | `127.0.0.1:8091` |
| infra-TAK | `127.0.0.1` | `8767` | `127.0.0.1:8767` |

Inner Docker nginx does **not** terminate TLS (`nginx/nginx.conf` serves HTTP on port 80 only). Do not map host port 443 into the container â€” use Caddy or host nginx for TLS.

Optional host-specific overrides: `docker-compose.override.yml` (gitignored).

## Authentik / OIDC expectations

infra-TAK auto-creates:

- **Provider name:** `EUD Remote Assist`
- **Slug:** `eud-remote-assist`
- **Client type:** `public` (not confidential)
- **Redirect URIs:**
  - `https://remote.<fqdn>`
  - `https://remote.<fqdn>/`
  - `https://remote.<fqdn>/callback`
  - `https://remote.<fqdn>/silent-callback`
- **Access:** `authentik Admins` group (infra-TAK admin console users)

The web app does **not** use `client_secret`. Device APIs use `X-Connection-Secret`, not Authentik.

## Versioning contract

infra-TAK uses this for update checks:

| Signal | Usage |
|--------|--------|
| `VERSION` (repo root) | Expected release after `git pull` |
| `GET /version` on **8448** | Running version post-deploy |
| `GET /health` on **8448** | Includes `version` field |
| Git tag `vX.Y.Z` | Release label matching `VERSION` |

Post-update check:

```bash
EXPECTED=$(cat VERSION)
RUNNING=$(curl -sS https://remote.example.com:8448/version | jq -r .version)
test "$EXPECTED" = "$RUNNING" && echo "OK" || echo "MISMATCH: want $EXPECTED got $RUNNING"
```

Loopback check (on host):

```bash
curl -sS http://127.0.0.1:8767/version
```
